# Sea Battle Bot

Telegram-бот для игры в Морской бой с соперником.

## Как играть

1. **Создать игру**: `/newgame` — бот выдаст код игры (например, `ABCDEF`)
2. **Пригласить соперника**: отправьте ему код
3. **Присоединиться**: `/join ABCDEF`
4. **Расставить корабли**: нажимайте на клетки доски, чтобы разместить корабль. После выбора нужного числа клеток нажмите "Подтвердить". Можно нажать "Расставить автоматически".
5. **Стрелять**: когда оба расставили корабли, ходите по очереди, нажимая на клетки доски соперника.

## Команды

| Команда | Описание |
|---------|----------|
| `/start` | Приветствие |
| `/newgame` | Создать новую игру |
| `/join XXXX` | Присоединиться по коду |
| `/leave` | Выйти из игры |
| `/cancel` | Отменить выбор при расстановке |

## Установка и запуск

### Локально

```bash
# 1. Клонировать репозиторий
cd sea_battle_bot

# 2. Установить зависимости
pip install -r requirements.txt

# 3. Получить токен бота у @BotFather в Telegram

# 4. Вписать токен в config.py
#    BOT_TOKEN = "ваш_токен"

# 5. Запустить
python main.py
```

### На сервере (Railway / Render / PythonAnywhere)

#### Railway (рекомендуется)

1. Залейте проект в GitHub
2. Создайте новый проект на [Railway](https://railway.app)
3. Подключите репозиторий
4. В настройках добавьте переменную окружения `BOT_TOKEN`
5. В `config.py` замените на:

```python
import os
BOT_TOKEN = os.getenv("BOT_TOKEN", "YOUR_BOT_TOKEN_HERE")
```

6. Railway сам установит зависимости и запустит `python main.py`

#### Render

1. Залейте проект на GitHub
2. Создайте новый Web Service на [Render](https://render.com)
3. Настройки:
   - Runtime: Python 3
   - Build Command: `pip install -r requirements.txt`
   - Start Command: `python main.py`
4. Добавьте переменную окружения `BOT_TOKEN`
5. Обновите `config.py` как указано выше для Railway

#### PythonAnywhere

1. Загрузите файлы через вкладку Files
2. Откройте Bash консоль
3. Установите зависимости: `pip install -r requirements.txt`
4. Настройте токен в `config.py`
5. Запустите: `python main.py` (будет работать пока открыта консоль)
6. Для фоновой работы используйте Always-on task (только на платных аккаунтах)
